## Folder and File Conventions

### Top-Level Folders

These folders organize your application's core code:

| **Folder** | **Description**             |
| ---------- | --------------------------- |
| `prisma`   | Database configuration      |
| `content`  | Documentation-related files |

### Top-Level Files

These files configure the application, manage dependencies, and define critical settings:

| **File**             | **Description**                                                                           |
| -------------------- | ----------------------------------------------------------------------------------------- |
| `tailwind.config.ts` | Tailwind CSS configuration file for themes, colors, plugins, and styles.                  |
| `sst.config.ts`      | SST (Serverless Stack) configuration file for deploying serverless infrastructure on AWS. |
| `source.config.ts`   | Manages documentation paths.                                                              |
| `components.json`    | Shadcn component library configuration.                                                   |

### Second-Level Files (`src`)

Files that provide core utilities for your application:

| **File**    | **Description**                                           |
| ----------- | --------------------------------------------------------- |
| `env.js`    | A type-safe file for importing environment variables.     |
| `config.ts` | Contains site-wide configuration, including Stripe plans. |

### Second-Level Folders (`src`)

Folders that organize auxiliary code:

| **Folder**   | **Description**                  |
| ------------ | -------------------------------- |
| `trpc`       | Handles tRPC client queries.     |
| `styles`     | Contains global styles.          |
| `server`     | Houses server utilities.         |
| `lib`        | Contains helper functions.       |
| `components` | Smaller UI components for reuse. |

### Server Files (`src/server`)

These files handle backend logic and operations:

| **File**         | **Description**                                                  |
| ---------------- | ---------------------------------------------------------------- |
| `api/trpc.ts`    | Contains tRPC procedures.                                        |
| `api/schemas.ts` | Defines reusable tRPC schemas.                                   |
| `api/root.ts`    | Exports all API routes.                                          |
| `api/error.ts`   | Provides a helper for throwing errors while preserving messages. |
| `api/routers/*`  | Contains route-specific logic based on database collections.     |
| `auth/index.ts`  | Exports default authentication functions.                        |
| `auth/config.ts` | Manages authentication providers and settings.                   |
| `db/index.ts`    | Initializes Prisma for both development and production.          |
| `db/queries.ts`  | Contains database query logic.                                   |
| `stripe.ts`      | Handles user subscription plans.                                 |

### Routing Files

Files that define routing logic and layouts:

| **File**            | **Description**                         |
| ------------------- | --------------------------------------- |
| `layout.tsx`        | Defines layout and SEO metadata.        |
| `error.tsx`         | Displays error UI.                      |
| `not-found.tsx`     | Displays "Not Found" UI.                |
| `layout.config.tsx` | Configuration for documentation layout. |

### Route Groups

Organized route folders based on user roles and states:

| **Folder**          | **Description**                   |
| ------------------- | --------------------------------- |
| `(admin)`           | Admin-specific routes.            |
| `(unauthenticated)` | Routes for unauthenticated users. |
| `(authenticated)`   | Routes for authenticated users.   |

### Nested Routes

Specific routes organized within functional folders:

| **Folder**                        | **Description**          |
| --------------------------------- | ------------------------ |
| `(admin)/tickets`                 | Manage all user tickets. |
| `(authenticated)/help`            | Open support tickets.    |
| `(authenticated)/settings`        | Manage global settings.  |
| `(unauthenticated)/auth`          | Sign-in page.            |
| `(unauthenticated)/docs`          | Documentation pages.     |
| `(unauthenticated)/legal`         | Legal documents.         |
| `(unauthenticated)/legal/privacy` | Privacy policy page.     |
| `(unauthenticated)/legal/terms`   | Terms of service page.   |
| `(unauthenticated)/pricing`       | Pricing page.            |
| `api`                             | API routes.              |
| `api/auth`                        | NextAuth route.          |
| `api/stripe-webhook`              | Stripe webhook handler.  |
| `api/trpc`                        | tRPC route.              |

### Private Folders

Folders reserved for reusable, large components:

| **Folder**    | **Description**                               |
| ------------- | --------------------------------------------- |
| `_components` | Large, reusable components used across pages. |

### SEO

Files and assets related to search engine optimization:

| **File**              | **Description**                                         |
| --------------------- | ------------------------------------------------------- |
| `layout.tsx`          | Contains SEO metadata.                                  |
| `sitemap.ts`          | Automatically generates a sitemap for your application. |
| `favicon.svg`         | Favicon for the website.                                |
| `opengraph-image.png` | Image displayed in link previews.                       |
